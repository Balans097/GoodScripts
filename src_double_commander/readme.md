
# Руководство по настройке Bash-сценариев для Double Commander

Это руководство описывает процесс установки, настройки и отладки пользовательских Bash-скриптов, интегрированных в панель инструментов Double Commander (DC). Все сценарии из набора используют единый надёжный механизм передачи файлов через параметр `%L`.

---

## 1. Подготовка среды

### 1.1. Где хранить скрипты
Рекомендуемое расположение для пользовательских исполняемых файлов в Linux:
```bash
~/.local/bin/
```
Если директории нет, создайте её:
```bash
mkdir -p ~/.local/bin
```
> 💡 Вы можете хранить скрипты в любой удобной папке. Главное — указывать **полный абсолютный путь** в настройках кнопки DC.

### 1.2. Права доступа
Каждый скрипт должен быть исполняемым:
```bash
chmod +x ~/.local/bin/имя_скрипта.sh
```

### 1.3. Системные зависимости
Сценарии используют внешние утилиты. Убедитесь, что они установлены (пример для Fedora):

| Назначение              | Пакет(ы)                  | Команда установки                  |
|-------------------------|---------------------------|------------------------------------|
| Изображения → PDF       | `img2pdf`                 | `sudo dnf install img2pdf`         |
| Видео (DAV → MP4)       | `ffmpeg`                  | `sudo dnf install ffmpeg`          |
| Документы → PDF         | `libreoffice`, `poppler-utils` | `sudo dnf install libreoffice poppler-utils` |
| Электронные книги       | `calibre`                 | `sudo dnf install calibre`         |
| DjVu → PDF              | `djvulibre`               | `sudo dnf install djvulibre`       |
| Разбиение PDF           | `qpdf`                    | `sudo dnf install qpdf`            |
| Определение типов файлов| `file`                    | `sudo dnf install file`            |

## 2. Настройка кнопки в Double Commander

1. Откройте: **Настройка → Настройка панели инструментов...**
2. Нажмите **Добавить** (`+`)
3. Заполните поля следующим образом:

| Поле в DC          | Что указывать                                      | Примечание |
|--------------------|----------------------------------------------------|------------|
| **Команда**        | Полный путь к скрипту<br>`/home/user/.local/bin/dcDav2mp4.sh` | Без кавычек. DC сам экранирует путь. |
| **Параметры**      | `%L`                                               | **Обязательно!** Передаёт временный файл со списком выделенных элементов. |
| **Путь запуска**   | `%D` или `%Ds`                                     | Директория **активной** панели. Поддерживается в этом поле. |
| **Подсказка**      | `DAV → MP4 (конвертация)`                          | Всплывающий текст при наведении. |
| **Файл значка**    | `video-x-generic` или оставить пустым              | Имя иконки из темы GTK. |

> ⚠️ **Важно:** В Double Commander нет отдельного поля «Вывод» для скрытия/показа консоли. Скрипты работают в фоне и используют `notify-send` для уведомлений.

## 3. Переменные Double Commander (актуальный справочник)

Переменные **чувствительны к регистру**. DC автоматически экранирует пути с пробелами (на Linux — экранирует backslash'ами).

| Переменная     | Значение                                                      | Пример (`/home/user/docs/book.pdf`) |
|----------------|---------------------------------------------------------------|-------------------------------------|
| **`%L`**       | Временный файл со **списком полных путей** выделенных элементов (по одному на строку) | `/tmp/_dc_~12345.lst` |
| **`%D`** / **`%Ds`** | Директория **активной (source)** панели (без завершающего слеша) | `/home/user/docs` |
| **`%P`** / **`%Ps`** | Полный путь к файлу **под курсором** (активная панель)      | `/home/user/docs/book.pdf` |
| **`%N`** / **`%fs`** | Имя файла под курсором (без пути)                            | `book.pdf` |
| **`%F`**       | Временный файл только с **именами файлов** (без путей)      | — |
| **`%S`**       | Выделенные файлы в одну строку через пробел (экранированные) | `book.pdf notes.txt` |

### Важные уточнения
- **`%L`** — основной и самый надёжный способ передать **все выделенные** файлы/папки в скрипт (рекомендуется для пакетной обработки).
- **`%D` = `%Ds`** — оба варианта идентичны. Используйте любой в поле «Путь запуска».
- **`%P` / `%Ps`** — относится только к файлу **под курсором**, а не ко всему выделению. Поэтому для скриптов, работающих с несколькими файлами, лучше использовать `%L`.

## 4. Как работает механизм `%L`

1. Выделяете файлы/папки.
2. Нажимаете кнопку.
3. Double Commander создаёт временный файл (например, `/tmp/_dc_~98765.lst`) и записывает в него **полные пути** (по одному на строку).
4. Запускает скрипт, передавая путь к этому файлу как первый аргумент: `скрипт.sh /tmp/_dc_~98765.lst`.
5. После завершения DC автоматически удаляет временный файл при выходе.

### Шаблон безопасного чтения `%L` (рекомендуется во всех скриптах)
```bash
LIST_FILE="${1:-}"
if [[ -z "$LIST_FILE" || ! -f "$LIST_FILE" ]]; then
    notify-send "Ошибка" "Не передан список файлов (%L)" -i dialog-warning
    exit 1
fi

# Очистка: убираем \r, лишние пробелы, кавычки, пустые строки
mapfile -t SELECTED < <(tr -d '\r' < "$LIST_FILE" | \
    sed 's/^[[:space:]]*//;s/[[:space:]]*$//;s/^"//;s/"$//' | \
    grep -v '^$')
```

## 5. Типичные ошибки и решения

| Симптомы                          | Причина                                      | Решение |
|-----------------------------------|----------------------------------------------|---------|
| `Permission denied`               | Скрипт не исполняемый                        | `chmod +x скрипт.sh` |
| `No such file or directory`       | Windows-переносы строк (CRLF)                | `dos2unix скрипт.sh` или сохранить как Unix (LF) |
| Скрипт запускается, но ничего не делает | В «Параметры» не указан `%L`                | Добавить `%L` |
| `command not found` (ffmpeg и т.д.) | Утилита не в PATH графической сессии        | Установить пакет + можно добавить `export PATH=...` в скрипт |
| Проблемы с пробелами/кириллицей   | Неправильное чтение списка                   | Использовать `mapfile + sed + tr` (как в шаблоне) |

## 6. Отладка сценариев

### Метод 1: Предпросмотр команды (`%?`)
Временно поставьте в **Параметры**:
```
%? %L
```
DC покажет итоговую команду и спросит подтверждение.

### Метод 2: Логирование
Добавьте в начало скрипта:
```bash
LOG="/tmp/dc_debug_$(date +%s).log"
exec > "$LOG" 2>&1
set -x
echo "[$(date)] Запуск: $0 $*"
```

### Метод 3: Запуск в терминале (для отладки)
Временно в поле **Команда**:
```bash
gnome-terminal -- bash -c "/полный/путь/к/скрипту.sh %L; exec bash"
```
