# Руководство по настройке Bash-сценариев для Double Commander

Это руководство описывает процесс установки, настройки и отладки пользовательских Bash-скриптов, интегрированных в панель инструментов Double Commander (DC). Все сценарии из набора используют единый надёжный механизм передачи файлов через параметр `%L`.

---

## 1. Подготовка среды

### 1.1. Где хранить скрипты
Рекомендуемое расположение для пользовательских исполняемых файлов в Linux:
```bash
~/.local/bin/
```
Если директории нет, создайте её и добавьте в `PATH` (обычно это делается автоматически в современных дистрибутивах):
```bash
mkdir -p ~/.local/bin
```
> 💡 Вы можете хранить скрипты в любой удобной папке. Главное — указывать **полный абсолютный путь** в настройках кнопки DC.

### 1.2. Права доступа
Каждый скрипт должен быть помечен как исполняемый:
```bash
chmod +x ~/.local/bin/имя_скрипта.sh
```

### 1.3. Системные зависимости
Сценарии используют внешние утилиты. Убедитесь, что они установлены:

| Назначение | Пакет(ы) | Команда (Fedora) |
|------------|----------|------------------|
| Изображения → PDF | `img2pdf` | `sudo dnf install img2pdf` |
| Видео (DAV → MP4) | `ffmpeg` | `sudo dnf install ffmpeg` |
| Документы → PDF | `libreoffice`, `poppler-utils` | `sudo dnf install libreoffice poppler-utils` |
| Электронные книги | `calibre` | `sudo dnf install calibre` |
| DjVu → PDF | `djvulibre` | `sudo dnf install djvulibre` |
| Разбиение PDF | `qpdf` | `sudo dnf install qpdf` |
| Определение типов файлов | `file` | `sudo dnf install file` |

## 2. Настройка кнопки в Double Commander

1. Откройте: **Настройка** → **Настройка панели инструментов...**
2. Нажмите **Добавить** (`+`)
3. Заполните поля **строго** следующим образом:

| Поле в DC | Что указывать | Примечание |
|-----------|---------------|------------|
| **Команда** | Полный путь к скрипту<br>`/home/user/.local/bin/dcDav2mp4.sh` | Без кавычек, даже если путь содержит пробелы. DC сам экранирует его. |
| **Параметры** | `%L` | **Ключевое поле!** Передаёт скрипту временный файл со списком выделенных элементов. |
| **Путь запуска** | `%D` или `%Ds` | Директория активной панели. `%P` здесь **не подходит** (это путь к файлу под курсором). |
| **Подсказка** | `DAV → MP4 (конвертация)` | Всплывающий текст при наведении. |
| **Файл значка** | `video-x-generic` | Имя иконки из текущей темы GTK. Можно оставить пустым. |

> ⚠️ **Важно:** В Double Commander **нет поля "Вывод" / "Output"** с опциями "Скрыто/Консоль". Все скрипты работают в фоне и показывают статус через системные уведомления (`notify-send`).

## 3. Переменные Double Commander (Справочник)

В полях **Команда**, **Параметры** и **Путь запуска** можно использовать макросы. Вот основные из них:

| Переменная | Значение | Пример для `/home/user/docs/book.pdf` |
|------------|----------|---------------------------------------|
| **`%L`** | Временный файл со списком выделенных путей (по одному на строку) | `/tmp/_dc_~12345.lst` |
| **`%D`** / **`%Ds`** | **Путь активной панели** (директория) | `/home/user/docs` |
| **`%P`** / **`%ps`** | **Полный путь к файлу под курсором** | `/home/user/docs/book.pdf` |
| **`%N`** / **`%fs`** | Имя файла под курсором (без пути) | `book.pdf` |
| **`%F`** | Список выделенных файлов (только имена) | Редко используется в Bash |
| **`%S`** | Выделенные файлы в одну строку через пробел | `book.pdf notes.txt` |

### 🔑 Разница между `%D` и `%P`
- `%D` — это **папка**, в которой вы сейчас находитесь. Используется в поле **Путь запуска**.
- `%P` — это **конкретный файл**, на котором стоит курсор. В наших сценариях не используется, так как мы работаем с выделением (`%L`).

## 4. Как работает механизм `%L`

1. Вы выделяете файлы или папки в одной из панелей DC.
2. Нажимаете кнопку.
3. DC создаёт скрытый временный файл (например, `/tmp/_dc_~98765.lst`) и записывает в него полные пути выделенных объектов.
4. DC запускает: `/путь/к/скрипту.sh /tmp/_dc_~98765.lst`
5. Скрипт читает путь к этому файлу из `$1`, открывает его и обрабатывает каждую строку.

### 📜 Шаблон безопасного чтения `%L` (используется во всех скриптах)
```bash
LIST_FILE="${1:-}"
if [[ -z "$LIST_FILE" || ! -f "$LIST_FILE" ]]; then
    notify-send "Ошибка" "Не передан список файлов (%L)" -i dialog-warning
    exit 0
fi

# Очистка от кавычек, Windows-переносов (\r) и пустых строк
mapfile -t SELECTED < <(tr -d '\r' < "$LIST_FILE" | sed 's/^[[:space:]]*//;s/[[:space:]]*$//;s/^"//;s/"$//' | grep -v '^$')
```

## 5. Типичные ошибки и их решение

| Симптомы | Причина | Решение |
|----------|---------|---------|
| `Permission denied` | Скрипт не помечен как исполняемый | `chmod +x /путь/к/скрипту.sh` |
| `No such file or directory` (при первом запуске) | В скрипте сохранены Windows-переносы строк (`CRLF`) | Откройте файл в редакторе → Сохранить как `Unix (LF)` или выполните `dos2unix скрипт.sh` |
| Скрипт запускается, но ничего не делает | В поле **Параметры** не указан `%L` | Вернитесь в настройки кнопки и впишите `%L` |
| `command not found` для утилит (`ffmpeg`, `img2pdf` и т.д.) | Утилита не установлена или отсутствует в `PATH` графической сессии | Установите пакет. В начале скриптов уже прописан `export PATH=...` для надёжности |
| Пути с пробелами/кириллицей ломают обработку | Неправильное чтение списка | Все наши скрипты используют `mapfile` + `sed` + `tr`, что полностью решает проблему пробелов |

## 6. Отладка сценариев

Поскольку скрипты работают в фоне, используйте эти методы для поиска ошибок:

### 🔹 Метод 1: Предпросмотр команды (`%?`)
В поле **Параметры** временно добавьте `%?` в начало:
```text
%? %L
```
При нажатии кнопки DC покажет итоговую командную строку и спросит подтверждение. Помогает убедиться, что пути передаются корректно.

### 🔹 Метод 2: Логирование в файл
Добавьте в начало скрипта (после `#!/usr/bin/env bash`):
```bash
LOG="/tmp/dc_debug_$(date +%s).log"
exec > "$LOG" 2>&1
set -x  # Включает трассировку каждой команды
echo "[$(date)] Запуск: $0 $*"
```
После нажатия кнопки откройте терминал и выполните: `tail -f /tmp/dc_debug_*.log`

### 🔹 Метод 3: Запуск в терминале (временно)
Замените поле **Команда** на:
```bash
gnome-terminal -- bash -c "/полный/путь/к/скрипту.sh %L; exec bash"
```
Это откроет окно терминала, покажет весь вывод и ошибки, и не закроется после завершения. После отладки верните исходный путь к скрипту.

## 7. Особенности предоставленных сценариев

Все скрипты следуют единой архитектуре:
- ✅ **Рекурсия и дедупликация:** Корректно обрабатывают смешанное выделение (файлы + папки), исключая повторную обработку одних и тех же файлов.
- ✅ **Безопасные пути:** Работают с кириллицей, пробелами и специальными символами.
- ✅ **Обратная связь:** Используют `notify-send` для вывода статуса в системный трей.
- ✅ **Автоматическая очистка:** Временные файлы и промежуточные артефакты удаляются автоматически.

| Сценарий | Что делает | Ключевая зависимость |
|----------|------------|----------------------|
| `dcImages2PDF.sh` | Объединяет изображения в один PDF (без потерь) | `img2pdf` |
| `dcDav2mp4.sh` | Конвертирует видео с камер в MP4 (без перекодирования) | `ffmpeg` |
| `dcDjvu2pdf.sh` | Пакетная конвертация DjVu/DJV в PDF | `djvulibre` |
| `dcDocs2PDF.sh` | Объединяет документы/электронные книги в один PDF | `libreoffice`, `calibre`, `poppler-utils` |
| `dcEpub2odt.sh` / `dcFb22odt.sh` | Конвертация книг в ODT через цепочку `DOCX → ODT` | `calibre`, `libreoffice` |
| `dcMerge2txt.sh` | Склеивает исходный код и текстовые файлы в один `.txt` | `file` |
| `Разбить pdf...qpdf.sh` | Делит большие PDF на части ≤ 19 МБ | `qpdf` |
