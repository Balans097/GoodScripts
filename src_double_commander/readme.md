
# Руководство по настройке сценариев для Double Commander

Это руководство описывает процесс создания, сохранения и подключения Bash-скриптов к кнопкам на панели инструментов Double Commander (DC).

---

## 1. Подготовка сценариев (Файловая система)

Все ваши скрипты (написанные на Bash) должны находиться в удобном месте и иметь права на запуск.

### 1.1. Где хранить скрипты?
Рекомендуемое место для пользовательских скриптов в Linux:
```bash
~/.local/bin/
```
Если папки нет, создайте её:
```bash
mkdir -p ~/.local/bin
```
*Примечание: Если вы используете другую папку (например, `~/Pool/Заметки/...`), убедитесь, что пути в настройках DC прописаны полностью.*

### 1.2. Права доступа
Скрипт должен быть исполняемым. В терминале выполните:
```bash
chmod +x ~/.local/bin/имя_сценария.sh
```

### 1.3. Зависимости
Убедитесь, что установлены системные утилиты, которые требуют скрипты:
*   **Для изображений:** `img2pdf`
*   **Для видео:** `ffmpeg`
*   **Для документов:** `libreoffice`, `calibre`, `poppler-utils` (для `pdfunite`/`pdfinfo`)
*   **Для DjVu:** `djvulibre`

## 2. Настройка кнопки в Double Commander

Чтобы запустить сценарий одной кнопкой:

1.  Откройте меню: **Настройка** → **Настройка панели инструментов...** (или `Configuration` → `Options` → `Toolbar`).
2.  Нажмите кнопку **Добавить** (`Add`).

### Заполнение полей конфигурации:

| Поле в окне DC | Что писать | Зачем это нужно |
| :--- | :--- | :--- |
| **Команда** (`Command`) | Полный путь к скрипту.<br>*Пример:* `/home/user/.local/bin/dcDav2mp4.sh` | Путь к исполняемому файлу скрипта. |
| **Параметры** (`Parameters`) | `%L` | **Самое важное!** Передает список выделенных файлов скрипту во временный файл. |
| **Путь запуска** (`Start path`) | `%P` | Запускает скрипт из активной папки DC. |
| **Подсказка** (`Tooltip`) | Текст (например, `DAV → MP4`) | Текст, который всплывает при наведении мыши на кнопку. |
| **Файл значка** (`Icon file`) | Имя иконки темы (например, `video-x-generic`) | Визуальный образ кнопки. Можно оставить пустым или выбрать из списка. |


---

## 3. Разбор переменных Double Commander

В поле **Параметры** можно использовать специальные переменные, которые DC подставляет автоматически:

| Переменная | Значение | Когда использовать |
| :--- | :--- | :--- |
| **`%L`** | **Список выделенных файлов.** DC создает временный файл, где в каждой строке записан полный путь к выделенному файлу. Этот путь передается в скрипт как `$1`. | **Для всех наших сценариев.** Позволяет обрабатывать сотни файлов и папки рекурсивно. |
| **`%P`** | Путь к активной панели. | Указывать в поле "Путь запуска". |
| **`%N`** | Имя файла под курсором (без пути). | Если скрипт работает только с одним конкретным файлом. |
| **`%F`** | Полный путь к файлу под курсором. | Аналог `%N`, но с полным путем. |
| **`%S`** | Выделенные файлы в одну строку через пробел. | **Не рекомендуется**, если в путях есть пробелы. Лучше используйте `%L`. |

---

## 4. Как это работает "под капотом"

1.  Вы выделяете файлы в Double Commander.
2.  Нажимаете кнопку.
3.  DC создает скрытый временный файл (например, `/tmp/_dc_~123.lst`) и записывает туда пути всех выделенных файлов.
4.  DC запускает команду: `/путь/к/скрипту.sh /путь/к/временному_файлу.lst`.
5.  Скрипт считывает этот файл (строка `while read -r line ... < "$1"`), обрабатывает данные и выдает результат.

## 5. Типичные ошибки и решение

### 1. Ошибка "No such file or directory" (Нет такого файла)
*   **Причина:** В скрипте сохранены Windows-переносы строк (`CRLF`), либо пробелы в путях не экранированы.
*   **Решение:**
    *   Откройте скрипт в редакторе и сохраните в формате **Unix (LF)**.
    *   Используйте в терминале: `dos2unix имя_скрипта.sh`.

### 2. Скрипт запускается, но ничего не делает
*   **Причина:** В настройках кнопки в поле **Параметры** ничего не написано.
*   **Решение:** Обязательно напишите `%L` в поле **Параметры**.

### 3. Ошибка "Permission denied"
*   **Причина:** Нет прав на запуск.
*   **Решение:** Выполните `chmod +x /полный/путь/к/скрипту.sh`.

### 4. Пути с пробелами ломают скрипт
*   **Решение:** Все наши сценарии настроены на чтение из `%L` построчно, что корректно обрабатывает пробелы и кириллицу в путях.
